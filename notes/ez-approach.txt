actual system design question
    - STEP 1: REQUIREMENTS (3-4 min)
        - Functional: "Kya kya hona chahiye?"
        - Non-functional: "Kilne users? Kitni speed?"
        - ASK questions. Dont assume.
        - Interviewer WANTS you to ask.
    - STEP 2: HIGH-LEVEL DESIGN (5-7 min)
        - Draw boxes Client, Server, Database
        - Identity core APIs (3-4 ondpoints)
        - Choose database type-SQL vs NoSQL (with reason)
    - STEP 3: DEEP DIVE (10-12 min)
        - Pick 2-3 components to go deeper
        - Add Caching layer (Redis)
        - Add Load Balancer
        - Add Message Queue for async tasks
        - Add CDN for static content
        - Discuss trade-offs "Maine YE choose kiya kyunki" (THIS impresses interviewers)
    - STEP 4: BOTTLENECKS & SCALE (3-5 min)
        - "Agar users 10x ho jayein tch?
        - Database sharding strategy
        - horizontal scaling approach
        - monitoring and failure handling
